China Technologies Newsletter in August, 2016

  • Shenzhen Stock Exchange migrated its trading system to Linux

        Shenzhen Stock Exchange upgraded its trading system to the next generation recently, they totally migrated the infrastructure from non-stop mainframe to X86 Linux clusters, the processing pattern is also turned from traditional centralized processing into distributed processing. For the new infrastructure, there are about 400 Linux machines deployed on multiple locations and data centers to support the FI, Equity, Fund and Derivative business and 300 million accounts. The software architecture of trading system is redesigned as well, they named it as ‘large-scale distributed parallel computation architecture on high-speed message bus’. After the combination of changes of infrastructure and software architecture, it gained a new generation trading system with better performance and 2/3 cost saving. [more…]

  • Baidu open sources its deep learning framework Paddle

        Baidu, a massive Chinese web company along the lines of Google, open sources its distributed deep learning platform Paddle and put it into github. As we know, Baidu invested much more on the cutting-edge technologies recently that is impacting its search and FinTech business. As they announced in official web site, Paddle is an open and easy-to-use deep learning platform for enterprise and research, it aims to provide a high effective framework for large scale deep learning and reduce the learning curve for developers. It is similar to Google TensorFlow but has more clear and clean source code and APIs, the performance is better than other competitors as well. [more…]

  • MicroServices Architecture and Distributed Tracing

        MicroServices architecture is a recent buzzword among software engineers. Many companies are refactoring(planning to refactor) their monolithic applications to microservices, it is like magic to attract thousands of developers on it. Many tier 1 and 2 Chinese internet services companies built the microservices framework against their needs, and even promote them to open source projects. The dubbo is the star among them and it has been widely used by many companies and applications.

        In general, there are many applications within a big organization, the dependency of applications is complicated, sometimes when a production issue is occurred, it is not easy to address where is wrong from the long end-to-end chains, so it is very important to build a distributed tracing system to collect, store and visualize all the traces. Google Dapper and Twitter Zipkin are just for it, but in China many companies are building their tracing system, such as taobao Eagle Eye, JD Hydra, dianping CAT and so on, it can help to make operation efficiently, especially for microservices architecture since that has more processes and machines.

  • The adoption of Apache Kylin™

        Apache Kylin™ is an open source Distributed Analytics Engine designed to provide SQL interface and multi-dimensional analysis (OLAP) on Hadoop supporting extremely large datasets. It is designed to reduce query latency on Hadoop for 10+ billion of rows of data, the users can interact with Hadoop data via Kylin at sub-second latency, better than Hive queries for the same dataset.

  • Case 1: Peking Mobile OLAP Cube

        Peking Mobile, the leading mobile telephony service provider generates 30 billion of rows of original data per day, the existing Hadoop cluster need to execute 800+ MapReduce jobs per day for data analysis that causes the cluster very busy.

    At the same time, the business analysts always want to run ad-hoc queries or generate reports, they tried to use Hive and Spark, but the Hive is too slow (1~2 hours) to satisfy the SLA and the Spark consumed too much memory, finally they choose Apache Kylin.

    They created some cubes from different dimensions via pre-defined jobs, it took about 1 hour for creating cubes on 30 billion of rows of data, most of queries can return the result within 20 seconds and the resource usage is in good shape, that is really a dramatic performance improvement compared to Hive.
